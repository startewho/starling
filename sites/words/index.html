This page is a small fixture for Starling. It deliberately uses the same handful of elements and CSS rules as a famously simple HTML page — nothing fancy, just markup for hierarchy and emphasis.

The point is coverage, not decoration.

You and the renderer connect because this markup was written and the layout engine read it. That's the web, reduced to one simple HTML page.

It weighs almost nothing. No framework, no build step: just paragraphs, a heading or two, a list, and a sprinkle of inline emphasis. Every tag on this page exists so a test can answer one question — "did this render?"

Elements exercised

  • a — anchor (with href, hreflang)
  • strong — strong emphasis
  • em — stress emphasis
  • span — inline grouping (with lang)
  • br — the line break above this list
  • ul / li — this very list
  • h1 / h2 — the two headings
  • p / p.note — the paragraphs, one styled
