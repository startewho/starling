Why build a new browser?

The common wisdom says a browser engine has to be C++ or Rust, built by a large team over many years. Ladybird already proved the hardest part — that a truly independent engine is possible. Starling shares that spirit but asks the next question: do you still need a systems language?

Starling is a managed-first .NET engine. The longer-term goal is to earn parity on real sites first, then use that managed core to make WebAssembly a first-class citizen — talking to the page directly instead of going through a JavaScript bridge.

Built from scratch

Own HTML parser, DOM, CSS, layout, paint, networking, and JavaScript engine. No third-party engine under the hood.

Managed-first .NET

A bet that a modern managed runtime can carry a real browser — safer code, faster iteration, fewer native shims.

Open and in the open

BSD 2-Clause licensed. Design docs, a public roadmap, and a work queue anyone can read.

Get started

You will need the .NET SDK 10.0.100 or newer. Clone with submodules, since Starling pulls in its regular expression engine from a sibling repo.

# Clone the repo with its submodules
git clone --recurse-submodules https://github.com/starling-browser/starling.git
cd starling

# Build and test
dotnet restore
dotnet build
dotnet test

# Render a live site straight to a PNG
dotnet run --project src/Starling.Headless -- render https://example.com -o example.png

The desktop browser runs through Aspire with aspire run. Full setup, including the free Six Labors paint license, is in the README.
